## Completed
- Moved `NmlPythonizer` into `pyneuroml/utils/nml2py.py` (renamed from `neuroml_pythonify.py`)
- Core algorithm: traverse object tree via `info(return_format="dict")`, generate `parent.add()` calls with resolved references
- Handles `NmlId` and `Nml2PopulationReferencePath` resolution to object references (e.g., `pop.id + "[0]"`)

- Components ordered by dependency: cells â†’ synapses â†’ channels â†’ inputs â†’ networks â†’ populations â†’ projections â†’ connections â†’ explicit inputs â†’ properties

## Key Design Decisions
- Root document created via `component_factory()`, all others via `parent.add()`
- Variable names derived from component `id` attributes
- Parent tracking via variable names (not id lookup) to avoid collisions when doc and network share same id
- Components without ids get type-based names with counters (e.g., `explicitinput_1`)
- `_is_scalar_type()` distinguishes attributes from child components using type patterns
- `_is_default_value()` compares against fresh instance defaults to avoid cluttering output
